Protocole HTTP

HyperText Transfer Protocol

Le protocole HTTP est tout simplement un langage texte qui permet à deux machines de communiquer ensemble.

Client envoie une requête

Une requête est un message texte envoyé par un client en respectant la syntaxe attendue par le protocole HTTP.

Voici un exemple de requête HTTP :


GET /mon-planning-semaine HTTP/1.1
Host: universite-savoie.com
Accept: text/html
User-Agent: Mozilla/5.0 (Windows NT 6.1; WOW64; rv:43.0) Gecko/20100101 Firefox/43.0
  • Méthode HTTP et URI
  • Entêtes de requête (format de réponses accepté par le client, informations sur le cache,...)

Serveur retourne une réponse

Voici un exemple de réponse HTTP :


HTTP/1.1 200 OK
Date: Sun, 07 Feb 2016 17:23:34 GMT
Server: Apache/2.4.10 (Debian)
Content-Type: text/html

Application web

L'objectif d'une application web reste toujours identique et ce avec n'importe quel langage de programmation.

Le principe fondamental du développement web est de comprendre chaque requête HTTP composée de texte et de retourner la réponse appropriée.

Requêtes et réponses en PHP


$uri = $_SERVER['REQUEST_URI'];
$foo = $_GET['foo'];
header('Content-type: text/html');
echo 'L\'URI demandée est: '.$uri;
echo 'La valeur du paramètre "foo" est: '.$foo;

PHP prépare les variables $_SERVER, $_GET,... qui contiennent les données de la requête ; le script exploite ces informations afin de créer une réponse.
